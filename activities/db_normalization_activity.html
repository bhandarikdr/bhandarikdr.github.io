Concept Overview What is Normalization?

Normalization means organizing a table to avoid repeated data and problems. We use three simple steps:

  1. 1NF -- No lists inside cells -- each cell has one value.
  2. 2NF -- Everything depends on the whole key -- no partial dependency.
  3. 3NF -- No chain dependencies -- no column depends on another non-key column.
Example: A table with Student_ID, Project_Name, Supervisor, Supervisor_Phone. The phone depends on the supervisor, not on the student. We need to split it.

The "Student Project" table (not normalized)

Task 1.1 -- What is wrong with this table? Look at the repeated data.

  1. Redundancy: Dr. Neupane's phone is repeated for every student.
  2. Update problem: If Dr. Neupane changes phone, you must update many rows.
  3. Delete problem: If Ram leaves, you lose Dr. Neupane's info entirely.

Conclusion: The table has update anomalies. We will fix them.

Part 2 Fix Step 1 & 2 (1NF and 2NF)

Step 1: Already in 1NF

Each cell has a single value (no lists), so it is already in 1NF. Now we go to 2NF.

Task 2.1 -- The primary key is Student_ID. But Supervisor_Phone depends on Supervisor, not on Student_ID. This is a partial dependency.

Task 2.2 -- Split the table into two tables to remove the partial dependency.

Table 1: Student (Student_ID, Student_Name, Project_Title, Supervisor)
Table 2: Supervisor (Supervisor, Supervisor_Phone)

Now Supervisor_Phone depends only on Supervisor (the whole key). The tables are in 2NF.

Part 3 Fix Step 3 (3NF)

Remove Transitive Dependency

Look at the Student table from Part 2: Student_ID to Supervisor. But what if we also store Supervisor_Department?

Student_IDStudent_NameProject_TitleSupervisorSupervisor_Dept
101Ram SharmaWeather AppDr. NeupaneComputer Science

Problem: Supervisor_Dept depends on Supervisor, not on Student_ID. That is a transitive dependency (Student_ID to Supervisor to Dept).

Task 3.1 -- Split again to reach 3NF.

Student (Student_ID, Student_Name, Project_Title, Supervisor)
Supervisor (Supervisor, Supervisor_Dept)

Now Supervisor_Dept depends on Supervisor, and Supervisor is a key in its own table. No transitive dependency. The tables are in 3NF.

Challenge Quiz: Test Your Understanding

1. What is the main goal of normalization?

  • To make tables larger
  • To reduce redundancy and avoid problems
  • To add more columns
Show answer
To reduce redundancy and avoid update/delete problems.

2. In the Student Project table, which dependency caused the 2NF violation?

  • Student_ID to Student_Name
  • Supervisor to Supervisor_Phone
  • Project_Title to Supervisor
Show answer
Supervisor to Supervisor_Phone (partial dependency on part of the key).

3. Which normal form removes transitive dependency?

  • 1NF
  • 2NF
  • 3NF
Show answer
3NF.
