Setup Environment Installation & Connection

Follow these steps to set up XAMPP and access the MySQL/MariaDB environment:

  1. Install XAMPP: Download the XAMPP installer executable package matching your host environment platform layout from the official Apache Friends portal. Complete the deployment setup package wizard utilizing default port options.
  2. Launch the Services: Open the XAMPP Control Panel from your desktop or Applications folder. Click the Start actions next to both the Apache and MySQL rows. Ensure their modules turn green.
  3. Navigate using Command Prompt (CLI): Launch your terminal interface shell environment (e.g., CMD on Windows) and run the change directory instructions sequence to jump directly into the standard location path: 
    cd C:\xampp\mysql\bin
  4. Login to MySQL/MariaDB Server: Authenticate securely into your local transactional database cluster using the administrative user account (root) with no default password assignment query rule: 
    mysql -u root
    Upon hitting Enter, the interface command cursor should shift to a live dashboard engine prompt state matching MariaDB [(none)]> or mysql>.

Activity Objective: Create a relational database schema structural engine for a library management system framework tracking inventory, and perform core CRUD (Create, Read, Update, Delete) procedural transactional instructions.

Part 1 Database and Table Setup

Step 1.1: Initialize a new database named LibraryDB inside your workspace cluster instance storage repository.

CREATE DATABASE LibraryDB;

Step 1.2: Verify and Select Database context to confirm presence and make it the active query operations target scope.

SHOW DATABASES;
USE LibraryDB;

Step 1.3: Create Books Table object infrastructure framework columns tracking core information variables.

CREATE TABLE Books (
    BookID int,
    Title varchar(255),
    Author varchar(255),
    PublishedYear int
);
Part 2 Data Manipulation (CRUD Operations)

Step 2.1: Insert Data record states securely to store data samples into your Books grid parameters table layout:

INSERT INTO Books (BookID, Title, Author, PublishedYear)
VALUES (1, 'The Great Gatsby', 'F. Scott Fitzgerald', 1925);

Step 2.2: Read Data rows matching query selections to fetch information back to the client interface layout:

Query all data details stored globally inside the table matrix context:

SELECT * FROM Books;

Query specific attribute keys selectively:

SELECT Title, Author FROM Books;

Step 2.3: Update Data profiles globally or conditionally targeting accurate unique row definitions safely using the conditional filtering clause tool:

UPDATE Books SET PublishedYear = 1926 WHERE BookID = 1;

Step 2.4: Delete Data profiles and isolate particular transactional elements using explicit row identifier matching specifications:

DELETE FROM Books WHERE BookID = 1;
Part 3 Advanced Structure & Cleanup Operations

Step 3.1: Modify Table Structure dynamically to add a new category property column labeled Genre seamlessly:

ALTER TABLE Books ADD Genre varchar(100);

Step 3.2: Index Management optimization initialization rules layout to speed up analytical target queries searching over titles:

CREATE INDEX idx_title ON Books(Title);

Step 3.3: Systematic Schema Cleanup routines implementation processing commands to drop entities from working environment allocations:

Remove operational tracking speed indexing configurations:

DROP INDEX idx_title ON Books;

Drop table structure and records completely from physical memory allocations:

DROP TABLE Books;

Drop entire logical structural database workspace tracking records context:

DROP DATABASE LibraryDB;
Challenge Practical Lab Tasks: CRUD Operations

Complete the following two hands-on challenge tasks in your MySQL terminal. Each challenge requires you to write and run code for all four CRUD operations.

Challenge Set 1: Student Enrollment Management

Assume you have a table named Students with the following columns: StudentID (int), FullName (varchar), Grade (int), and Section (varchar). Write the SQL statements to complete these operations:

  • 1. Create (Insert): Add a new student named 'Anil Sharma', who is in Grade 11, Section 'A' with a StudentID of 101.
  • 2. Read (Select): Write a query to display only the FullName and Section of all students who are in Grade 11.
  • 3. Update: Modify the record to change Anil Sharma's section from 'A' to 'B'.
  • 4. Delete: Remove the student from the table whose StudentID is exactly 101.

Challenge Set 2: Store Inventory Control

Assume you have a table named Inventory with the following columns: ItemID (int), ItemName (varchar), Price (int), and StockQuantity (int). Write the SQL statements to complete these operations:

  • 1. Create (Insert): Add a new item named 'Wireless Mouse' with an ItemID of 5, a price of 1200, and a stock quantity of 15.
  • 2. Read (Select): Write a query to list all details for items in the inventory that have a StockQuantity of less than 20.
  • 3. Update: Increase the stock quantity of the 'Wireless Mouse' (ItemID 5) by adding 10 more items (changing the total stock value to 25).
  • 4. Delete: Remove the item from the inventory completely where the ItemID is 5.
View Challenge Solutions

Expected Syntax Solutions for Verification:

[Set 1 Solutions]

-- 1. Create
INSERT INTO Students (StudentID, FullName, Grade, Section) VALUES (101, 'Anil Sharma', 11, 'A');
-- 2. Read
SELECT FullName, Section FROM Students WHERE Grade = 11;
-- 3. Update
UPDATE Students SET Section = 'B' WHERE StudentID = 101;
-- 4. Delete
DELETE FROM Students WHERE StudentID = 101;

[Set 2 Solutions]

-- 1. Create
INSERT INTO Inventory (ItemID, ItemName, Price, StockQuantity) VALUES (5, 'Wireless Mouse', 1200, 15);
-- 2. Read
SELECT * FROM Inventory WHERE StockQuantity < 20;
-- 3. Update
UPDATE Inventory SET StockQuantity = 25 WHERE ItemID = 5;
-- 4. Delete
DELETE FROM Inventory WHERE ItemID = 5;
