🐞 Looking back · code review

WHAT

Did I focus only on getting code to compile, or did I strive to understand theory & complexity? Which concepts (arrays, functions, recursion, file handling) did I truly master? Was I a stack‑overflow copy‑paster or did I use pseudocode? How did I debug? How did I manage screen time and balance?

WHY

What drove me? Passing tests, parental expectations, or genuine curiosity? How did that motivation affect my frustration when code wouldn’t compile or logic failed?

⚙️ Moving forward · system design

WHAT

Which topics (OOP, advanced data structures, SQL, capstone) spark curiosity or intimidation? What technical milestones, clean‑coding standards, and grade targets? How do these connect to university, tech career, or problem‑solving?

HOW

Which messy habits (poor naming, no comments, coding without plan) must I leave behind? Which industry practices will I adopt? How will I structure weekly deep‑focus sessions while protecting well‑being? What’s my proactive plan for mentorship, collaboration, and systematic debugging?

WHY

What is my “Big Why” beyond the grade? What kind of system thinker, digital creator do I want to become? How will this deeper purpose preserve resilience when code breaks and deadlines loom?
